# World Cup 2026 AI Analytics Command Center

> BI dashboards, match prediction, upset-risk modeling, and NLP fan sentiment analysis powered by Python, SQL, and scikit-learn.

![World Cup 2026 AI Analytics Command Center](assets/dashboard-preview.png)

An end-to-end analytics project built on ~49,000 international matches that takes
raw data all the way to business-style insight across **three integrated layers**:

1. **BI dashboard** — match results, group standings, goals, and team KPIs, powered by SQL.
2. **ML model** — win / draw / loss prediction with an **upset-risk** score for every upcoming fixture.
3. **NLP dashboard** — fan sentiment by team from a custom, dependency-free sentiment engine.

Everything is wired into one self-contained, interactive **Command Center**
(`outputs/command_center.html`) — open it in any browser, no server required.

```
 data collection ──▶ cleaning ──▶ SQL marts ──▶ ┌─ BI dashboard
 (results.csv)       (pandas)     (SQLite)       ├─ ML predictions + upset risk
                                  feature eng. ──┤
                                  + sentiment ───┴─ NLP sentiment
                                                    └▶ command_center.html
```

---

## The three sections

**1. BI / Overview.** A SQLite star schema (`fact_match` + `dim_team`) loaded from cleaned
data; analytical SQL (`sql/queries.sql`) computes tournament KPIs, group standings, goal
trends, and dominant results. Each of the 48 teams also carries a strength index (Elo) and
recent-form KPIs.

**2. ML / Predictions.** Match-outcome models predict win/draw/loss from Elo, recent form,
rolling goals, rest days, venue, and match context. Evaluated on a **time-based split**
(train ≤2017, test 2018–2025):

| Model | Accuracy | Log-loss |
|---|---|---|
| Naive baseline (always home) | 0.478 | 1.051 |
| Elo-only baseline | 0.601 | 0.873 |
| Full-feature logistic | 0.600 | 0.867 |
| Blend (deployed) | 0.598 | 0.868 |

**The honest reading:** every model strongly beats the naive home-win baseline (~60% vs 48%
accuracy, ~0.87 vs 1.05 log-loss). The engineered features add a *small* improvement in
probability quality (log-loss) over the Elo-only baseline, while **accuracy stays around 60%** —
the blend does **not** meaningfully beat the Elo baseline on accuracy. Elo is the dominant
signal; the extra features refine the probabilities rather than the headline accuracy. The
probabilities are well-calibrated, and an **upset-risk** metric (= probability the higher-Elo
favorite fails to win) surfaces the fixtures most likely to produce a shock.

> Exact numbers vary slightly with each data refresh; `outputs/metrics.json` always holds the
> figures for your current run.

**3. NLP / Sentiment.** A self-contained, VADER-style analyzer (`src/sentiment.py`) —
sentiment lexicon + negation + intensifier handling — scores fan comments to a compound
value in `[-1, 1]` and aggregates per-team sentiment KPIs.
> ⚠️ **The fan corpus is synthetic / illustrative** — generated text whose tone is biased by
> each team's real form, clearly labelled as such in the dashboard so no one mistakes it for
> scraped social data. Swap in real data by replacing `build_corpus()` with a loader for a
> `team,text` export from the Reddit API (PRAW), a Kaggle football-tweets dataset, or the
> YouTube comments API — the scoring and aggregation run unchanged, and that upgrade would
> make the NLP section materially stronger.

## Run it

```bash
pip install -r requirements.txt
./run_all.sh          # downloads data, runs every stage, builds the dashboard
# then open outputs/command_center.html
```

> **Note:** the first full training run may take a few minutes because it trains the models
> and computes permutation importance. Use `refresh.sh` for faster dashboard updates after the
> model is trained (it skips retraining).

### Live mode

The match dataset updates continuously, so the project is "live" in the sense that
it reflects whatever results are current the moment you rebuild. Played matches move
out of the predictions list into a **results table**, and the model **grades itself**
against them in a live scorecard (it's a strictly pre-tournament model — trained only
on matches before June 11 — so the grading has no leakage).

To pull the latest results and rebuild in one step:

```bash
./refresh.sh          # re-downloads results.csv, re-runs ETL -> predict -> sentiment -> dashboard
```

For true hands-off updating you'd host the dashboard and run `refresh.sh` on a schedule
(e.g. a cron job or GitHub Action); a local file is refreshed on demand.


Or step by step:

```bash
python src/etl.py             # clean -> SQLite -> KPI marts + bi.json
python src/train.py           # train/evaluate ML model -> model.joblib + metrics + plots
python src/predict.py         # match predictions + upset risk -> ml.json
python src/sentiment.py       # fan sentiment by team -> nlp.json
python src/build_dashboard.py # assemble outputs/command_center.html
```

One-off prediction:
```bash
python src/predict.py   # all WC2026 fixtures
```

## Honest notes

- **No leakage:** every ML feature is computed in a single chronological pass, reading each
  team's state *before* kickoff. The model is evaluated on a forward time split, not random folds.
- **Elo is the dominant signal;** added features give a small, real improvement — a useful
  finding in itself.
- **The sentiment corpus is synthetic/illustrative;** the pipeline is real and ready for live data.
- **Data:** martj42 / *International football results from 1872 to 2017* (CC0), kept current.
